# RefPlayer

アニメーション・イラスト制作のリファレンス用ローカル動画プレイヤーです。  
フレーム単位の操作、範囲切り抜き書き出し、オニオンスキンなどの機能を備えています。

---

## 機能

| 機能 | 説明 |
|---|---|
| フレームステップ | 1フレーム単位で前後に移動 |
| 再生速度（FPS）切り替え | 表示FPSをリアルタイムに変更 |
| イン/アウト点設定 | 範囲を指定してループ再生・書き出し |
| MP4書き出し | 指定範囲をMP4に書き出し（HWエンコーダ自動選択） |
| タイムラインズーム | 細かいフレーム操作に対応 |
| オニオンスキン | 前後フレームを半透明合成で表示（ビルトインプラグイン） |
| ドローイングタイマー | 時間制限付き練習用タイマー（ビルトインプラグイン） |
| ユーザープラグイン | JS ファイルを置くだけで機能を拡張できる |

## ダウンロード / インストール

`dist/` フォルダに以下が生成されます（`npm run package` 実行後）：

| ファイル | 説明 |
|---|---|
| `RefPlayer Setup 3.3.0.exe` | インストーラー版（推奨） |
| `RefPlayer 3.3.0.exe` | ポータブル版（インストール不要） |

インストーラー版を実行し、画面の指示に従ってください。

## 使い方

### 動画を開く
- ヘッダーの **「ファイルを開く」** ボタン → 動画ファイルを選択
- 対応形式：MP4, MOV, AVI など（Chromium が再生できる形式）

### 再生・フレーム操作
- **スペース**：再生 / 一時停止
- **← →**：1フレーム後退 / 前進
- **r**：リピート切り替え

### 範囲切り抜き・書き出し
1. **`[`** でイン点、**`]`** でアウト点を設定
2. **Enter** で範囲をクリップ（タイムラインが切り抜き範囲に絞られる）
3. ヘッダーの **「MP4書き出し」** ボタン → 保存先を指定
4. 書き出し中はプログレスバーが表示される

### タイムラインのズーム
- **Ctrl + =**：ズームイン
- **Ctrl + -**：ズームアウト

### 設定画面
- **Ctrl + ,**：設定を開く
- ショートカットキーをカスタマイズできます

| 操作 | キー |
|---|---|
| 再生 / 一時停止 | `Space` |

## プラグイン

### ビルトインプラグイン

アプリに最初から組み込まれています。サイドバー（右上のパズルアイコン）で表示できます。

| プラグイン | 説明 |
|---|---|
| **オニオンスキン** | 前後のフレームを合成モードで半透明表示。前後フレーム数・不透明度を調整可能 |
| **ドローイングタイマー** | 15s / 30s / 60s の時間制限練習用タイマー。円グラフで残り時間を表示 |

### ユーザープラグインの追加

1. **`Ctrl + ,`** で設定を開き、「プラグインフォルダを開く」ボタンを押す
2. 開いた `%APPDATA%\RefPlayer\plugins\` フォルダに、プラグインのフォルダごとコピーする
3. アプリを再起動するとサイドバーに追加される

**フォルダ構成の例：**
```
%APPDATA%\RefPlayer\plugins\
└── my-plugin\
    ├── manifest.json
    └── index.js
```

サンプルプラグインは `examples/plugins/hello-world/` を参照してください。  
プラグインの作り方は `examples/plugins/hello-world/README.md` に詳しく書かれています。

## ソースからビルド

### 必要環境

- [Node.js](https://nodejs.org/) v18 以上
- Windows 10/11（`npm run package` は Windows 向けビルドのみ対応）

### セットアップ

```bash
git clone <このリポジトリ>
cd RefPlayer
npm install
```

### 開発モードで起動

```bash
npm run dev
```

### 配布用ビルド

> Windows の「開発者モード」を有効にしてから実行してください。  
> （設定 → システム → 開発者向け → 開発者モード：ON）

```bash
npm run package
```

`dist/` に `.exe` インストーラーとポータブル版が生成されます。

新バージョンを公式リリースする場合の手順は [docs/RELEASE.md](docs/RELEASE.md) を参照してください。

## セキュリティ

### プラグインの取り扱い

ユーザープラグイン（`%APPDATA%\RefPlayer\plugins\` 内のJSファイル）は、RefPlayer の renderer プロセスの **完全な権限** で実行されます。これは次のことを意味します：

- DOM 操作・Canvas 描画・動画フレームへのアクセスができる
- `window` グローバル経由でアプリの内部 API（`window.refplayer.*`）にもアクセスできる
- ローカルファイルの読み書きや任意のネットワーク通信が可能

このため、**信頼できる作者のプラグインのみを導入してください**。不明なソースから入手したプラグインは絶対に `plugins/` フォルダに配置しないでください。

プラグインを審査するときの目安：
- ソースコード（`index.js`）を読んで、明らかに不審な処理（外部サーバーへの通信、ファイルアップロード等）がないか確認する
- `manifest.json` の `id` が他のプラグインと衝突していないか
- 公式または信頼できる配布元から入手したものか

### 脆弱性の報告

セキュリティ上の問題を発見した場合は、公開 Issue ではなく非公開の方法（リポジトリの「Security」タブから Security Advisory を作成、または直接メール）で報告してください。

### 配布版バイナリの署名

現在配布している `RefPlayer Setup *.exe` / `RefPlayer *.exe` は **コードサインされていません**。インストール時に Windows SmartScreen の警告が表示されますが、これは署名証明書がないためで、悪意のあるバイナリということではありません。「詳細情報」→「実行」で進めることができます。
